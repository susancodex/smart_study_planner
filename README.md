# Smart Study Planner

Smart Study Planner is a Django REST Framework API for managing study plans and tasks with JWT authentication, per-user permissions, searchable/filterable task endpoints, and autogenerated API docs.

## Features

- User registration and JWT login
- Study plan CRUD endpoints scoped to the authenticated user
- Task CRUD endpoints scoped to the authenticated user
- Study plan progress endpoint
- Search and filter support for tasks
- OpenAPI schema and Swagger UI via drf-spectacular
- Automated tests for auth and ownership permissions
- Docker and Render deployment support

## Tech Stack

- Python 3.13
- Django 6
- Django REST Framework
- djangorestframework-simplejwt
- django-filter
- drf-spectacular
- Whitenoise
- Gunicorn
- PostgreSQL on Render, SQLite locally

## Main API Endpoints

Authentication:

- `POST /api/auth/register/` - create a user
- `POST /api/auth/login/` - obtain JWT access and refresh tokens
- `POST /api/auth/token/refresh/` - refresh an access token

Planner:

- `GET /api/planner/studyplans/` - list study plans for the authenticated user
- `POST /api/planner/studyplans/` - create a study plan
- `GET /api/planner/studyplans/<id>/` - retrieve a study plan
- `PUT/PATCH /api/planner/studyplans/<id>/` - update a study plan
- `DELETE /api/planner/studyplans/<id>/` - delete a study plan
- `GET /api/planner/studyplans/<id>/progress/` - view completion percentage
- `GET /api/planner/tasks/` - list tasks for the authenticated user
- `POST /api/planner/tasks/` - create a task
- `GET /api/planner/tasks/<id>/` - retrieve a task
- `PUT/PATCH /api/planner/tasks/<id>/` - update a task
- `DELETE /api/planner/tasks/<id>/` - delete a task

Docs:

- Swagger UI: `/api/docs/`
- OpenAPI schema: `/api/schema/`

## Local Development

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

To create a superuser locally:

```bash
python manage.py createsuperuser
```

Run tests:

```bash
python manage.py test --verbosity=2
```

## Environment Variables

Copy `.env.example` and set these values before deploying:

- `DJANGO_SECRET_KEY`
- `DJANGO_DEBUG`
- `DJANGO_ALLOWED_HOSTS`
- `DATABASE_URL`

## Render Deployment

This repo includes a `Dockerfile`, `Procfile`, and `render.yaml` for Render.

1. Push the repository to GitHub.
2. Create a new Render Web Service connected to this repo.
3. Use Docker deployment or import `render.yaml`.
4. Set the environment variables above in Render.
5. Add a Render PostgreSQL database and connect it through `DATABASE_URL`.
6. After deploy, run:

```bash
python manage.py migrate
python manage.py collectstatic --noinput
```

## Notes

- Study plans and tasks are user-scoped, so one user cannot access another user's records.
- The project uses SQLite by default locally and PostgreSQL when `DATABASE_URL` is present.
